不是很严谨的结果：

  • 10 个列表项，遍历一直更快；
  • 100 个，slice 略微胜出；
  • 1,000 个，slice 比遍历快了 40%~69%；
  • 10,000 个，slice 快了 33%~50%；
  • 100,000 个，slice 快了 40%~50%；
  • 1,000,000 个，二者各有胜负；1
  • 10,000,000 个，filter 略微胜出。2

结论

数量不多时，filter 和 slice 差别不大。除非要处理成千上万的列表项，否则还是写更容易让人理解和维护的代码为好。

如果你的项目里面需要处理相当数量的记录，还请告诉我是不是有最佳实践。

译者注

1 这篇文章是 2016 年，距现在已经 3 年了。在 node v14.15.0 使用同样的脚本测试，1,000,000 列表项，slice 领先 filter 可达到 60% 左右。

2 10,000,000 列表项，node v14.15.0 slice 领先 filter 可达到 50% 左右。
